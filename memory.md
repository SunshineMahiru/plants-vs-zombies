## Recent Changes
- 修正小推车资源路径，并调整显示比例，避免明显变形
- 修正向日葵卡片费用为 `50`
- 修正寒冰射手卡片费用为 `175`
- 增加 `PlantAttackUtil.hasZombieInLane(int plantX, int plantY)`，统一处理射手同排目标检测
- 同排判定已过滤：
  - 只检测同一行
  - 只检测仍存活、未进入死亡动画的僵尸
  - 只检测植物右侧的僵尸
- 修正阳光回收移动逻辑：
  - 改为固定朝左上角目标点收敛
  - 避免点击后卡在左上角旋转不消失
  - 自然下落速度改为至少每帧移动 1 像素，避免快到终点时停住
- 修正双射手发射时机：
  - 同排一出现有效僵尸即可进入发射节奏
  - 同排无有效僵尸时停止持续发射
- 针对“寒冰正常、豌豆不发射”再次回退双射手发射条件：
  - 统一改回 `hasTarget && (!this.attack || shootInterval >= 1.5)`
  - 不再使用 `new Date(0)` 方式重置计时
  - 让首次发现目标立即发射，之后按固定间隔继续发射
- 修正僵尸行坐标生成：
  - 新增 `ROW_TOP_Y`
  - 僵尸改为按行顶部 `y` 生成，不再直接用行中心 `y`
  - 目标是修复最下面一行偶发错位、导致本行植物打不到的问题
- 进一步修正僵尸行对齐根因：
  - 新增 `getZombieSpriteY(row, bodyHeight)`
  - 生成僵尸时按“僵尸碰撞体中心 = 草坪该行中心”反推图片绘制 `y`
  - `PlantAttackUtil` 也改为按僵尸碰撞体中心判定所属行
  - 目标是解决“僵尸显示在两行中间、一路吃到相邻两行植物”的严重错位问题
- 修正坚果动画逻辑：
  - 不再按高血/中血/低血三段循环
  - 改为按血量比例映射 `jianguo` 的 `1-115` 帧序列
  - 活着时按当前血量显示对应帧
  - 血量到 0 时显示最后一帧，第 `115` 帧显示后移除植物
- 补充开场右拉镜头展示：
  - `IntroAnimation` 不再只展示旗帜僵尸
  - 现在会随机展示旗帜、铁桶、报纸、橄榄球 4 类僵尸
- 修正开场右拉镜头时机：
  - 右拉过程中也开始绘制预览僵尸
  - 不再等镜头完全拉到右侧停住后才突然出现
- 补充多阶段僵尸血量比例分段：
  - 铁桶、报纸、橄榄球不再写死用 `100` 作为高低血切换阈值
  - 统一改为 `maxHp / 2`
  - 这样同一套素材逻辑可以随僵尸总血量变化而保持比例切换
- 修正式式刷怪随机性：
  - 新增 `spawnRandomZombie(row)`
  - 从第一波开始就随机生成旗帜、铁桶、报纸、橄榄球
  - 不再出现前面长时间只刷旗帜、打完才刷别的类型
- 修正橄榄球僵尸命中框：
  - 加宽碰撞框，覆盖主要躯干
  - 用于修复子弹穿过橄榄球僵尸却不命中的问题
- 修正射手开火判定：
  - 目标改为“本行且已经进入当前可见画面中的僵尸”
  - 不再因为僵尸还在屏幕外就提前进入攻击状态
  - 本行僵尸死亡或离开当前可见画面后会退出攻击状态
- 修正铲子与菜单冲突：
  - 点击铲子区域改为进入铲除模式
  - 不再直接打开暂停/menu 页面
  - 铲除成功后自动退出铲子拖动状态
- 清理 `Flower` 中误混入的无关攻击状态残留

## Current Issues
- 本轮未实际启动游戏验证，只完成了代码层修正
- `GameObject.java` 仍然过大，后续维护成本高
- 当前植物/僵尸创建逻辑还不是简单工厂结构，新增类型时仍会改到多处
- 当前暂停/menu 的单独入口仍不明确，如果老师要求暂停按钮，后续需要单独补一个明确入口

## Refactor Direction
- 如果继续重构，优先按职责拆分 `src\plantgame`
- 推荐拆分方向：
  - `plantgame/core`：`GameObject`、通用工具、基础常量
  - `plantgame/plants`：向日葵、豌豆、寒冰、坚果
  - `plantgame/zombies`：各类僵尸
  - `plantgame/projectiles`：子弹、阳光、小推车
  - `plantgame/factory`：植物工厂、僵尸工厂
- 目标：满足老师提到的“简单工厂模式”，新增植物或僵尸时尽量少改旧代码
