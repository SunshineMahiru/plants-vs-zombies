# 《植物大战僵尸》Java复刻版项目解读

本项目是一个使用Java AWT/Swing技术栈实现的《植物大战僵尸》复刻版游戏。项目结构清晰，采用了典型的面向对象设计，通过自定义游戏主循环、双缓冲渲染以及多线程技术实现了流畅的游戏体验。

## 1. 项目结构总览

项目主要包含以下几个关键目录：
- `src/`：存放所有的Java源文件和游戏资源（图片、音效等）。
- `bin/` / `out/`：编译后的class文件输出目录。
- `src/plantgame/`：核心代码包，包含了游戏的所有逻辑实现。

资源文件夹分类非常细致：
- `Zombies/`：存放各类僵尸（旗帜、铁桶、报纸、橄榄球等）的序列帧图片。
- `flower/`, `wandou/`, `hanbing/`, `jianguo/`：存放各类植物的图片和GIF。
- `sounds/`：存放游戏的BGM和各种音效（如射击、啃咬、收集阳光等）。
- `GameFrame/`, `MenuFrame/`, `LoadFrame/`等：存放各个游戏界面的背景和UI元素。

## 2. 核心架构设计

游戏采用了基于状态机的界面切换机制，所有的界面都通过统一的渲染管线进行绘制。

### 2.1 游戏入口与主循环 (`Enter.java`)
`Enter` 类是整个游戏的入口，继承自 `java.awt.Frame`。
- **主窗体设置**：创建了一个 900x600 的固定大小窗口。
- **双缓冲渲染**：为了防止画面闪烁，重写了 `update` 方法，使用 `offScreenImage` 在内存中先绘制好一帧，然后一次性贴到屏幕上。
- **游戏主循环**：内部类 `HelpRepaint` 是一个不断循环的线程，每 40 毫秒（即 25 FPS）调用一次 `repaint()`，驱动整个游戏的画面更新。
- **事件分发**：通过监听鼠标的点击和移动事件，根据当前处于哪个界面（`LoadFrame.live`, `MenuFrame.live`, `GameFrame.live` 等标志位），将事件分发给对应的界面类处理。

### 2.2 游戏对象基类 (`GameObject.java`)
`GameObject` 是所有游戏中实体（植物、僵尸、子弹、阳光、小推车等）的基类。
- **共有属性**：坐标(`x`, `y`)、宽高(`wide`, `high`)、速度(`speed`)、图像(`img`)、存活状态(`live`)。
- **核心方法**：
  - `drawSelf(Graphics g)`：绘制自身。
  - `getRect()`：返回对象的包围盒（Rectangle），用于后续的碰撞检测。

基于 `GameObject` 派生出了丰富的子类：
- **植物类**：`Flower`（向日葵）、`Wandou`（豌豆射手）、`Hanbing`（寒冰射手）、`Jianguo`（坚果）。
- **僵尸类**：`QiZhi`（旗帜僵尸）、`TieTong`（铁桶僵尸）、`BaoZhi`（报纸僵尸）、`GanLan`（橄榄球僵尸）。
- **其他实体**：`Sun`（阳光）、`Bullet`（子弹）、`Car`（小推车）、`Card`（植物卡片）、`Shove`（铲子）等。

### 2.3 游戏核心逻辑 (`GameFrame.java`)
`GameFrame` 是游戏最核心的类，负责管理战斗场景的所有逻辑。
- **场景管理**：定义了草坪的网格坐标计算（5行9列），以及镜头的平移（`cameraX`），实现了类似原版中背景比屏幕大、可以左右移动的效果。
- **对象池/列表**：使用多个 `ArrayList` 管理场景中的各类对象（`glass`网格、`zombie`列表、`plant`列表、`bullet`列表等）。
- **核心逻辑更新（在 draw 方法中顺带处理）**：
  - **绘制**：按层级绘制背景、植物、僵尸、子弹、UI（工具栏、阳光数）。
  - **生成逻辑**：根据游戏时间（`loadtime` 和 `startTime`）自动生成天空掉落的阳光和随机出现的僵尸。
  - **植物生产**：向日葵定时生产阳光。
  - **冷却机制**：管理植物卡片、铲子、锤子等工具的冷却时间。
- **交互逻辑**：处理鼠标的点击（种植植物、收集阳光、使用铲子/锤子）和移动（卡片跟随鼠标）。

### 2.4 工具类 (`GameUtil.java`)
封装了底层的资源加载和通用计算：
- **图片加载**：`getImage` 读取普通图片，`getGifFrames` 解析 GIF 动画文件为 `Image[]` 数组，方便逐帧绘制动画。
- **音频播放**：使用 `javax.sound.sampled` 包实现 BGM 的循环播放和短促音效的播放。
- **碰撞检测**：提供 `ifRect` 方法判断点是否在矩形内。

## 3. 关键机制解析

### 3.1 碰撞检测与伤害结算
在实体类的 `drawSelf` 方法中（如 `QiZhi.drawSelf`）不仅处理了绘制，还处理了自身的逻辑更新：
- **子弹击中僵尸**：遍历 `GameFrame.bullet` 列表，使用 `getRect().intersects()` 判断子弹包围盒是否与僵尸包围盒相交。如果相交，则扣除僵尸血量，移除子弹，播放音效。如果是寒冰子弹，还会修改僵尸的 `speed` 实现减速。
- **僵尸啃食植物**：遍历植物列表，检测碰撞。如果碰到植物，僵尸停止移动（`moving=false`，`attacking=true`），记录开始攻击的时间。利用时间差（每隔1秒）对植物造成伤害。当植物血量归零时，恢复僵尸的移动状态。

### 3.2 动画实现
游戏没有使用复杂的动画引擎，而是通过**序列帧**来实现：
- 资源被解析为 `Image[] imgs` 数组。
- 每个对象维护一个 `count` 变量。
- 在每次 `drawSelf` 时，绘制 `imgs[count]`，然后 `count++`。
- 对于僵尸，甚至通过 `count` 的范围来区分状态（如 0-47 是高血量行走，48-95 是高血量攻击，96-143 是低血量行走等）。

### 3.3 镜头平移
通过 `java.awt.Graphics` 的 `translate(x, y)` 方法实现。
在绘制游戏场景前，调用 `g.translate(-offsetX, 0)`，然后绘制所有基于背景绝对坐标的游戏对象。绘制完场景后，再 `g.translate(offsetX, 0)` 恢复坐标系，接着绘制固定在屏幕上的 UI（如工具栏、阳光数量）。

## 4. 总结
该项目是一个完成度较高的 Java 课设级别的游戏。它抛弃了复杂的第三方游戏框架，完全基于 Java SE 标准库（AWT/Swing/Sound）从零构建了游戏循环、渲染管线、事件系统和实体状态机。代码虽然将逻辑更新和绘制耦合在了一起（都写在 `drawSelf` 中），但整体逻辑直观，非常适合用来学习面向对象设计和基础的 2D 游戏开发原理。
